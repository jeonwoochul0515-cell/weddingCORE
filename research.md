### 2.4 개인정보보호법과의 충돌

## 5. 이용자 피해 및 신뢰도 문제

### 5.1 외국인 배우자 피해

- 결혼이민자 17만 명 중 **가정폭력과 혼인단절이 최대 문제**
- F-6-1 비자 연장 시 **남편 협조 필수** → 체류권이 배우자에 종속
- 가정폭력 피해 시 F-6-2(자녀양육)/F-6-3(혼인단절) 전환 심사 극히 까다로움
- 인권 전문가: "소득 높은 쪽에서 후진국 여성을 사는 형태, 인신매매와 다를 바 없다"
- 여성의 얼굴/나이/키/몸무게 등 신체조건 공개, 품평, 등급 매기는 관행 잔존

### 5.2 한국인 이용자 피해

- **무등록 중개업자 사기**: 중개비 1,000만 원+ 수령 후 잠적
- **허위 신상정보**: 나이, 학력, 혼인 이력 허위 전달
- **추가 비용 요구**: 계약 외 추가비용, 계약 외 지불방법 요구
- **환불 거부**: 중개 실패에도 환불 거부 (분쟁 68.4%)
- 평균 1,372만 원 투입 후 결혼 실패 시 경제적 타격 심각

### 5.3 업계 신뢰도 위기

- **"매매혼" 이미지 고착**: 학계/시민단체/언론에서 지속 비판
- **외교 마찰**: 2006년 베트남 현지 맞선 보도 → 양국 외교 위기
- **법 위반 만연**: 3년간 75건 위반, 자정 능력 의문
- 미디어 보도: "국제결혼중개업 광고 속 여성은 '물건'이었다"

### 5.4 코로나19 이후 변화

- **비대면 연애 기반 결혼비자 신청 급증** → 매매혼 감소 빈자리 대체
- 화상 맞선/온라인 상담이 업계 표준으로 정착
- **문제점**: 진정성 입증 어려움, "결혼 전 현지 방문" 검증 장치 무력화
- 허위 프로필, 사진 조작 등 온라인 사기 위험 증가

## 9. Phase 1.5: 매칭 보조 도구 (신규 추가)

> **배경**: Phase 1이 컴플라이언스에 올인되어, 중개업소가 매일 쓰는 "매칭 실무 도구"가 부재.
> 컴플라이언스 = 안 걸리려고 하는 것(방어), 매칭 도구 = 돈 버는 것(공격).
> **돈 버는 도구가 없으면 구독 동기가 약하다.**

### 9.1 Phase 1.5 스코프 (~4주)

| 기능 | 설명 | 기간 | Firestore 위치 |
|------|------|------|---------------|
| **G1. 고객 프로필 관리** | 기본 인적사항(나이/직업/학력/종교/지역) + MBTI + 생년월일 구조화 입력/조회 | 1주 | `agencies/{agencyId}/clients/{clientId}` 필드 확장 |
| **G2. MBTI 궁합** | 16x16 매트릭스 궁합 점수 + 간단 해설 UI | 0.5주 | 프론트엔드 전용 (정적 매트릭스) |
| **G3. 사주 궁합 (라이트)** | 생년월일(양력/음력) 기반 간단 궁합 점수, "재미로 보는" 톤 | 1주 | 프론트엔드 전용 (룰 기반 계산) |
| **G4. 매칭 히스토리** | 누구와 맞선했는지, 결과(수락/거절/보류), 매니저 메모 | 0.5주 | `agencies/{agencyId}/matches/{matchId}` 신규 컬렉션 |
| **G5. 매칭 제안 리포트** | 양측 프로필 비교 요약 1페이지 PDF 출력 | 1주 | PDF 생성 (클라이언트 사이드) |

### 9.2 G1. 고객 프로필 관리 — 상세

기존 `clients/{clientId}` 문서에 매칭용 필드를 확장:

```
clients/{clientId}
├── (기존) name, nationality, birthDate, ...
├── (신규) profile: {
│     mbti: "ENFP",
│     height: 165,
│     weight: 55,
│     education: "대졸",
│     occupation: "간호사",
│     religion: "무교",
│     region: "부산",
│     smoking: false,
│     drinking: "가끔",
│     hobbies: ["요리", "여행"],
│     idealType: {
│       ageRange: [30, 45],
│       mbtiPrefer: ["ISTJ", "ISFJ"],
│       regionPrefer: ["부산", "경남"],
│       ...
│     }
│   }
└── (신규) birthInfo: {
      solarDate: "1990-03-15",
      lunarDate: "1990-02-19",
      birthTime: "오시"  // 사주용 (선택)
    }
```

### 9.3 G2. MBTI 궁합 — 상세

- **구현**: 프론트엔드 전용, 정적 16x16 궁합 매트릭스
- **점수 체계**: 1~100점 + 궁합 등급 (천생연분 / 좋음 / 보통 / 노력 필요)
- **해설**: 각 조합별 1~2줄 해설 (예: "ENFP × ISTJ: 서로의 부족한 면을 채워주는 보완적 관계")
- **UI**: 매칭 제안 시 양측 MBTI 배지 + 궁합 점수 + 해설 카드

```
궁합 매트릭스 예시:
         ISTJ  ISFJ  INFJ  INTJ  ISTP  ISFP  ...
ENFP      85    72    90    78    65    80   ...
ENTP      70    60    82    92    75    65   ...
...
```

### 9.4 G3. 사주 궁합 (라이트) — 상세

**포지셔닝**: "재미로 보는 궁합" — 매칭 결정의 주 도구가 아닌 **참고 콘텐츠**

- **입력**: 양측 생년월일(양력) + 성별 (최소), 태어난 시간(선택)
- **산출**: 천간/지지 기반 간단 궁합 점수 (오행 상생/상극)
- **출력**: 궁합 점수(1~100) + 한줄 해설 + 오행 시각화
- **면책**: "본 궁합은 전통 명리학을 참고한 재미 콘텐츠이며, 실제 관계에 대한 예측이나 보장이 아닙니다" 명시

**구현 방식**:
- 양력→음력 변환: `korean-lunar-calendar` npm 패키지
- 천간/지지 계산: 생년월일 → 사주팔자(년주, 월주, 일주, 시주) 추출
- 궁합 로직: 오행 상생/상극 + 천간합/지지합 룰 기반
- 프론트엔드 전용 계산, 서버 불필요

**리스크 관리**:
- "과학적 근거 없는 미신" 비판 → "재미" 톤 + 면책 고지
- 결혼이민자 차별 시비 → 매칭 결정 근거로 사용 불가 안내, UI상 보조 위치

## 10. weddingCORE 전략적 시사점

### 10.1 포지셔닝

```
"국제결혼중개업 올인원 운영 SaaS — 컴플라이언스 자동화 + 매칭 보조 + 법적 증거 생성"
```

- 국내외 유일한 포지션 — **퍼스트무버 기회**
- 가장 유사한 모델: Envoy Global (이민법 SaaS) + Boundless (배우자비자 자동화)
- 변호사법 회피: "법률 자문"이 아닌 **"행정 자동화 도구"**
- 창희 변호사의 법률 전문성 + 우철의 기술 = **진입 장벽**
- Phase 1.5 추가로 **"돈 버는 도구"** 확보 → 구독 동기 강화

### 10.2 타겟 고객

- **1차 타겟**: 전국 370~400개 등록 국제결혼중개업소
- **초기 공략**: 부산/경남 지역 (창희 변호사 인적 네트워크 활용)
- **확장**: 국내결혼정보회사 → 동남아 → 일본/대만 유사 규제 시장

### 10.3 프라이싱 전략

| 티어 | 월 구독료 | 포함 기능 |
|------|----------|----------|
| Basic | 5~8만 원 | 체크리스트, 서류보관, 유효기간 알림, **프로필 관리, MBTI 궁합** |
| Pro | 10~15만 원 | Basic + 비자 시뮬레이터, 이중언어 UI, 규제 레이더, **사주 궁합, 매칭 리포트 PDF** |
| Enterprise | 별도 협의 | Pro + 변호사 상담 연동, API 연동 |
| 변호사 상담 | 건당 과금 | 30:70 (플랫폼:청송) |

참고 벤치마크: 캐시노트(자영업자 매출관리) 성공 모델 — 저가 월정액 + 높은 리텐션

### 10.4 고객 획득 전략

1. **업종 협회 파트너십**: 결혼중개업협회 연계
2. **지자체/감독기관 연계**: "컴플라이언스 준수율 향상" 명분
3. **콘텐츠 마케팅**: 규제 가이드, 체크리스트 무료 배포 → 리드 확보
4. **레퍼럴**: 기존 고객 추천 프로그램
5. **창희 변호사 네트워크**: 부산/경남 법조 인맥 활용

### 10.5 기술 우선순위

| 순위 | 기술 | Phase | 근거 |
|------|------|-------|------|
| 1 | 룰 엔진 (json-rules-engine) | 1 | 핵심 가치 — 체크리스트/시뮬레이터 |
| 2 | 알림 자동화 (카카오 알림톡) | 1 | 유효기간 만료, 법령 변경 알림 |
| 3 | 다국어 문서 템플릿 | 1 | 이중언어 병렬 합의 UI |
| 4 | AI OCR (CLOVA OCR) | 2 | 신분증/서류 자동 인식 |
| 5 | 자동 번역 (Papago API) | 2 | 한↔베/한↔중 번역 초안 |
| 6 | 전자서명 (카카오 전자서명) | 2 | 중개계약서/동의서 전자 수집 |
| 7 | AI 챗봇/RAG | 3 | 규제 Q&A 자동 처리 |

### 10.6 핵심 차별화 (세일즈 포인트)

1. **"행정처분 방지 보험"** — 체크리스트 하나로 40+ 의무항목 누락 제로
2. **세계 최초** 이중언어 병렬 합의 UI + F-6 비자 시뮬레이터
3. **MBTI + 사주 궁합** — 매니저가 고객에게 보여주는 시각적 매칭 근거
4. **변호사 백업** — 복잡 케이스는 가족법 전문 변호사에게 바로 에스컬레이션
5. **영세 업소도 쓸 수 있는 가격** — 월 구독형, 초기 비용 없음
6. **투명성 제고** — 프로세스 기록으로 윤리적 중개 입증

### 10.7 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| 시장 규모 한계 (400개소) | 국내결혼 확장 + 해외 진출로 TAM 확대 |
| 영세 업소 지불 능력 | 프리미엄 또는 건당 과금으로 진입 장벽 최소화 |
| 규제 변화 속도 | 규제 레이더(기능 D)로 오히려 경쟁 우위 |
| 업계 축소 추세 | 생존 업체의 컴플라이언스 니즈는 오히려 증가 |
| 변호사법 저촉 | "자동화 도구" 포지셔닝, 법률 판단은 룰 코드화 |
