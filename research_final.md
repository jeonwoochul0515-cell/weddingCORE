## 3. 법적 리스크 분석

### 3.1 결혼중개업법 의무사항

| 의무 | 위반 시 제재 |
|------|-------------|
| 신상정보 제공 (혼인경력/건강/범죄/직업/종교/학력) | 3년 징역 / 3,000만 원 |
| 공증 (양국 공증인 인증, 아포스티유/영사확인) | 영업정지 → 등록취소 |
| 통번역 (이해 가능한 언어로 작성) | 1회 3개월, 2회 6개월, 3회 등록취소 |
| 거짓/과장 광고 금지 | 2년 징역 / 1,000만 원 |
| 명의대여 금지 | 3년 징역 / 3,000만 원 |
| 개인정보 보호 | 3년 징역 / 3,000만 원 |
| 동시 소개 / 미성년자 소개 금지 | 3년 징역 / 3,000만 원 |

### 3.2 등록 요건

- 자본금: **1억 원 이상** (계속 유지)
- 보증보험: **5,000만 원 이상**
- 사전교육: 한국건강가정진흥원 이수
- 등록 관청: 시/군/구청

### 3.3 개인정보보호법 충돌

결혼중개업법이 특별법 지위로 예외 적용되나, 동의서 취득 절차 미비/중개 실패 후 파기 미이행/현지 브로커 유출 시 양법 동시 위반 가능.

### 3.4 국제사법 / 출입국관리법

- 혼인 성립요건: 각 당사자 본국법 (국제사법 제63조)
- 허위 초청: 3년 징역 / 2,000만 원
- 위장결혼 알선: 결혼중개업법 + 출입국관리법 동시 적용
- 초청 횟수: 5년 내 2회 이상 시 사증발급 불허

## 4. 현장 인터뷰 (누나)

**대상**: 배선영 소장 / 공인중개사(태양 공인)

### 4.1 핵심 데이터

| 항목 | 답변 |
|------|------|
| 관리 도구 | **수기 노트** (엑셀도 안 씀) |
| 월 접촉 고객 | **~30명** |
| 월 성사 | **1팀** (3.3%) |
| 모객 채널 | **교차로 광고** |
| 제일 힘든 점 | **상담영업 (모객)** |
| 서류 관리 | 행정사에 외주, 별로 신경 안 씀 |
| 유효기간 | "놓쳐도 다시 받으면 되지" |
| 가격 민감도 | **월 5만 원 이상 안 냄** |
| 전체 소감 | "편리할 것 같은데, 오래된 업소는 잘 모르겠음" |

### 4.2 매칭 관련

- 매칭 기준: **외모 1순위**, 나이 안 따지고 직업/경제력 더 중요
- 프로필: 현지 에이전시에서 여성 5명 프로필 받아서 남성에게 보여줌
- 사진+가족관계/학력/신체지수/직업 함께 적어서 보냄
- MBTI/사주: 한국 남성은 신경 안 씀, **베트남이 사주 많이 봄**

### 4.3 현지 에이전시 구조

- **파트너 관계** (직원 아님), 4개 팀 거래 (라오스 2, 일본 1, 우즈벡 1)
- 에이전시 역할: 여성 모집, 서류 전부 총괄, 여학당 기숙 관리
- 비용: **500~700만 원** (총 수수료의 절반 가까이)
- 비자 관련 서류: **전부 현지 에이전시가 총괄**

### 4.4 소통 방식

- 베트남 여성: Zalo 쓰지만, 맞선보러 올 때 **카톡 설치 + 번역기로 소통 가능**
- 현지 에이전시: **카톡 사용**
- 맞선 후 남녀 카톡으로 직접 연락

### 4.5 시사점

| 발견 | 전략 영향 |
|------|----------|
| 1위 페인포인트 = 모객 | 컴플라이언스보다 모객 지원이 급함 |
| 월 5만 원 캡 | SaaS 구독만으론 수익 불충분 |
| 서류는 행정사가 처리 | 서류 관리 도구 가치 낮음 (현재로선) |
| 오래된 업소는 관심 낮음 | 초기 타겟 = 신규/초보 업소 |
| 카톡 번역으로 소통 가능 | 자체 번역 채팅 불필요 |

## 5. 번역 채팅 검토 및 기각

### 5.1 검토 결과

| 항목 | 분석 |
|------|------|
| 시장 니즈 | 있음 (비대면 교제 증가, F-6 증빙 필요) |
| 기술 비용 | 월 ~$41 (10업소 기준), 번역 API가 85% |
| 법적 리스크 | 중간 (통신비밀보호법, 개인정보보호법) |
| 개발 기간 | 3~4주 |

### 5.2 기각 사유

> **카카오톡 번역이 무료이고 이미 양쪽 다 설치되어 있다.**

누나 인터뷰에서 직접 확인: "베트남 여성들은 카톡 설치해서 번역키로 소통에 문제없어함"

토큰 비용 써가면서 무료 카톡과 경쟁할 이유 없음. → **Phase 2에서 제거**

### 5.3 대안

카톡을 그대로 쓰게 하되, 카톡이 못 하는 것만 해결:
- F-6 비자 서류 자동 수집/생성
- 국가별 법률 가이드
- 매칭 기록/프로필 관리

## 6. B2B2B 피봇 분석

### 6.1 현실의 소통 구조

```
한국 업소 ←→ 매니저 ←→ 현지 브로커 ←→ 외국 여성
(카톡)       (수동중계)   (Zalo/WeChat)  (수동적)
```

### 6.2 외국 여성 직접 접근성

| 국가 | 직접 접근 가능? | 이유 |
|------|---------------|------|
| 베트남 | **X** → △ | Zalo 주로 쓰나 카톡 설치 가능 (에이전트 세팅) |
| 중국 | **X** | Firebase **GFW 차단**, 접속 자체 불가 |
| 캄보디아 | **X** | 디지털 리터러시 최저 |
| 필리핀 | △ | 영어 가능, 상대적 나은 편 |
| 우즈벡 | **X** | Telegram 중심, 한국 웹 접근 경험 없음 |

### 6.3 핵심 발견

> **현지 브로커가 여성 측의 모든 디지털 행위를 대행한다** — 앱 설치, 계정 생성, 번역 중계, 사진 수집, 영상통화 세팅까지.

### 6.4 피봇 방향 (검토 후 보류)

```
한국 업소(B) ═══ weddingCORE ═══ 현지 브로커(B) ═══ 외국 여성
```

→ 그러나 **CODEF API 발견 이후** 전략이 다시 변경됨 (섹션 10 참조)

## 8. 법률서비스 + AI 결합 모델

### 8.1 국제결혼 단계별 법률 이슈 → AI 제공 가능 범위

| 단계 | 고객이 검색하는 것 | AI 안전 제공 (무료) | 전환 트리거 |
|------|------------------|-------------------|-----------|
| 탐색 | "국제결혼 절차", "비용" | 국가별 A-Z 가이드, 비용 해설 | "견적 받기" |
| 업체 선택 | "사기 업체 구별법" | 등록번호 조회, 체크리스트 | "인증 업체 소개" |
| 계약 | "계약서 정상인가" | 표준계약서 vs 내 계약서 비교 | "전문가 상담" |
| 비자 | "F-6 조건", "소득 미달" | **F-6 시뮬레이터 + CODEF 연동** | "서류 자동 수집" |
| 정착 | "귀화 조건", "체류 연장" | 요건 체크리스트 | "행정사 연결" |
| 분쟁 | "이혼 시 비자", "사기 배상" | 일반적 권리 안내 | **창희 변호사 상담** |

### 8.2 변호사법 안전 설계

2025년 서울고법 판결: "표준화된 질의 기반 AI 정보 제공"은 합법.

| 안전 | 위험 (금지) |
|------|-----------|
| "F-6 비자에 필요한 서류 목록입니다" | ~~"귀하의 경우 비자 승인 가능성이 높습니다"~~ |
| "표준계약서에는 이런 조항이 포함됩니다" | ~~"이 계약서 조항은 불리하니 수정하세요"~~ |
| 무료 제공 | ~~유료 법률 상담 (비변호사)~~ |

모든 AI 응답 하단에: "법률사무소 청송 감수 | 일반적 법률 정보이며 법률 자문이 아닙니다"

### 8.3 AI대륜 사례 교훈

법무법인 대륜의 AI 챗봇 → 대한변협 징계 청구 → 8개월 만에 중단. 법무법인이 직접 운영해도 리스크 존재. → **"법률 정보"와 "법률 자문"의 경계를 절대 넘지 않는 설계 필수**.

## 9. 수익 모델 / 가격 정책

### 9.1 3층 케이크 모델

```
┌─────────────────────────────────────────────┐
│  3층: 법률 상담 (건당)                        │
│  AI+변호사 하이브리드: 3~5만 원 (30:70 배분)    │
│  변호사 직접 상담: 10~15만 원 (30:70)          │
│  소송 수임: 200만+ (변호사 100%)               │
├─────────────────────────────────────────────┤
│  2층: 서류 자동화 (건당) ← CODEF 핵심 수익원    │
│  F-6 서류 패키지: 30~50만 원                   │
│  (행정사 100만 원 → 반값 자동화)                │
├─────────────────────────────────────────────┤
│  1층: SaaS 구독 (월정액) ← 진입 도구           │
│  Basic: 3만 원 (CRM + 컴플라이언스)            │
│  Standard: 5만 원 (+ 리드 포함)               │
└─────────────────────────────────────────────┘
```

### 9.2 왜 SaaS만으로 안 되는가

- 전국 업소 400개 × 월 5만 원 (100% 점유) = 월 2,000만 원 (비현실적)
- 현실적 초기: 40개 × 5만 원 = **월 200만 원** (부족)
- **서류 자동화(건당 30~50만)가 실제 수익원**

### 9.3 리드 가치 산정

```
리드 가치 = 업소 마진(300~900만) × 전환율(10~20%)
         = 30만~180만 원/리드
→ 리드당 과금: 5~30만 원 (업소 마진의 1~5%)
```

### 9.4 3면 플랫폼 가격 설계

| 사이드 | 과금 | 이유 |
|--------|------|------|
| 남성 고객 | **무료** (보조금 사이드) | 리드의 원천, 유입시켜야 네트워크 효과 |
| 변호사 | **무료** (참여비 0원) | 1명(창희), 수임 건에서 수익 |
| 중개업소 | **과금 사이드** | 리드가 곧 매출 (건당 1,372만 원) |

### 9.5 단계별 가격 로드맵

| 단계 | 남성 고객 | 업소 | 변호사 |
|------|----------|------|--------|
| 0. 베타 (~6개월) | 무료 AI 법률정보 | 무료 베타 (5개소) | 무료 |
| 1. 초기 수익화 | 무료 + 유료상담 | 월 3만 + 리드 5만 | 30:70 배분 |

## 10. CODEF API 연동 — 게임 체인저

### 10.1 배경

우철님이 이미 CODEF API 17개를 LawDocs(개인회생 서류 자동생성)와 law-caddy(변호사 사무실 SaaS)에 연동 완료한 상태.

### 10.2 CODEF API 17개 → F-6 비자 매핑

| API # | API명 | F-6 비자 용도 | 적용 가능 |
|-------|-------|-------------|----------|
| #1 | 은행 보유계좌 조회 | 예금 잔액 → 재산 증빙 | **가능** |
| #3 | 대출거래내역 조회 | 부채 현황 파악 | **가능** |
| #4 | 저축은행 보유계좌 | 추가 예금 잔액 | **가능** |
| #7 | 카드 대출내역 | 카드론/리볼빙 부채 | **가능** |
| #9 | 보험 계약정보 | 해지환급금 → 자산 | **가능** |
| #10 | 증권 전계좌 | 증권 보유 → 자산 | **가능** |
| #12 | 부동산 공시가격 | 주거 요건 + 자산 증빙 | **가능** |
| #13 | 자동차등록원부 | 차량 자산 | **가능** |
| #14 | 지방세 부과내역 | 체납 확인 | **가능** |
| #15 | **소득금액증명** | **F-6 핵심 — 소득 요건** | **가능** |
| #16 | 납세증명(체납조회) | 국세 체납 확인 | **가능** |
| #17 | **건강보험 자격득실** | 직장/지역 구분, 가구원 수 | **가능** |

**17개 중 12개가 F-6 비자 심사에 직접 활용 가능.**

### 10.3 이것이 바꾸는 것

```
현재 남성 고객:
  홈택스 → 주민센터 → 건보공단 → 은행 → 법원 → 등기소
  6~8곳 돌아다님, 1~2주 소요

weddingCORE + CODEF:
  공인인증서 1회 인증 → 3분 만에 전부 수집
  → "캄보디아는 월 $2,500 필요, 당신은 월 320만원 → 통과"
  → 필요 서류 패키지 자동 생성
```

### 10.4 비즈니스 모델 전환

```
AS-WAS: 정보 제공 플랫폼 (콘텐츠)
TO-BE:  서류 자동화 플랫폼 (실행)
```

**"국제결혼 서류, 공인인증서 한 번이면 끝"**

헬프미가 법인설립에서 한 것과 같은 구조:
- 행정사가 100만 원 받는 것을 AI+API로 반값(30~50만 원)에
- 결혼중개업 등록 없이 가능 (서류 대행은 중개가 아님)
- 창희 변호사 법률 감수가 경쟁 우위
- LawDocs에서 이미 구현한 기술 재활용 가능

### 10.5 확인 필요 사항

1. CODEF에 weddingCORE용 별도 서비스 등록 필요 여부
2. 개인정보 동의 절차 (본인 공인인증서 + 동의)
3. CODEF API 사용료 구조
4. F-6 비자 서류 자동 생성이 행정사법/변호사법에 저촉되는지
