# ⚠ 법률 검수 가이드 (창희 변호사 최종 검토용)

## 배경

`functions/src/admin/obligationTemplates.ts`에 40개 의무 항목 초안이 시드되어 있습니다.
이 데이터는 **AI(Claude)가 공개 법령을 기반으로 작성한 초안**이며,
실제 서비스 운영 전 반드시 창희 변호사의 법률 검수가 필요합니다.

## 검수 우선순위

### 🔴 즉시 검수 필요 (critical 등급 16개)
플랫폼이 이들 항목을 "완료"로 표시하는 순간 업체의 행정처분 방어 자료로 쓰이므로
법조문 번호·제재 범위가 정확해야 합니다.

- REG_VALID_CHECK, INSURANCE_CHECK (등록/보험)
- CONTRACT_WRITTEN (서면계약)
- BG_KOR_MARITAL_HISTORY, BG_KOR_HEALTH, BG_KOR_CRIMINAL
- BG_PARTNER_MARITAL_HISTORY, BG_PARTNER_HEALTH, BG_PARTNER_CRIMINAL
- BG_TRANSLATION, BG_MUTUAL_DELIVERY
- VISA_INCOME_CHECK, VISA_SINCERITY_EVIDENCE, VISA_PROGRAM_COMPLETED

### 🟡 중점 검토 필요 (high 등급)
제재 수위·기한 계산 공식을 실무 관례와 비교.

### 🟢 참고 검토 (medium / low 등급)
전반적 논리 일관성만 검토.

## 검수 체크 포인트 (각 항목당)

1. **법조문 번호 정확성**: `legalBasis` 필드 - 조문 번호·항·호 확인
2. **제재 수위 정확성**: `penaltyRange` 필드 - 행정처분 기준표 대비
3. **기한 계산 공식**: `dueDateRule` - 실제 실무상 적정 여부
4. **필수 증빙**: `requiredEvidence` - 누락·과잉 확인
5. **중요도 등급**: `severity` - critical/high/medium/low 재분류

## 검수 후 반영 방법

1. 변호사가 수정안을 Google Docs 또는 직접 코드로 전달
2. 우철이 `obligationTemplates.ts` 파일 업데이트
3. 관리자 계정으로 `/admin/agencies` → "의무 템플릿 시드" 버튼 클릭
4. 기존 문서는 `merge: true`로 업서트되므로 데이터 손실 없음
5. 검수 이력은 `updatedBy`, `updatedAt` 필드에 자동 기록

## 확인 완료 선언

검수 완료 시 Firestore `obligationTemplates/_meta` 문서 생성:
```
{
  version: "1.0.0",
  reviewedBy: "창희 변호사 (법률사무소 청송)",
  reviewedAt: Timestamp,
  barNumber: "...",
  notes: "전체 40개 항목 법률 검수 완료"
}
```

이 메타 문서가 있어야 UI에 "법률 검수 완료" 배지가 표시되도록 추후 연동 예정.
